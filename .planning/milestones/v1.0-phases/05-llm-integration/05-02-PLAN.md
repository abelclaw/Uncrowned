**HybridParser** (`src/game/llm/HybridParser.ts`):

Create a class that wraps TextParser and the LLM layer (OllamaClient, PromptBuilder, ResponseMapper).

Constructor: creates instances of TextParser, OllamaClient, PromptBuilder, ResponseMapper internally. In the constructor, call `this.ollamaClient.checkAvailability()` as a fire-and-forget promise so availability is known by the time the player types their first command.

`async parse(input, hotspots, exits, inventoryItems?, roomContext?): Promise<ParseResult>` method:

1. Try `this.textParser.parse(input, hotspots, exits, inventoryItems)` -- this is synchronous and instant.
2. If `regexResult.success === true`, return it immediately. The regex path is the fast path and handles all well-formed commands.
3. If regex failed AND `this.ollamaClient.isAvailable() === false`, return the regex error result (graceful fallback -- no LLM available).
4. If regex failed AND Ollama is available, try the LLM path:
   a. Build prompt via `this.promptBuilder.build(input, hotspots mapped to { id, name }, exits mapped to { id, direction, label }, inventoryItems ?? [], roomContext ?? { name: 'unknown' })`
   b. Call `this.ollamaClient.chat(prompt.systemPrompt, prompt.userMessage, GAME_ACTION_SCHEMA)`
   c. Map response via `this.responseMapper.map(response.content, hotspots, exits, inventoryItems ?? [], input)`
   d. Return the mapped ParseResult
5. If the LLM path throws (timeout, network error, invalid response, abort), catch the error, `console.warn('LLM parse failed, using regex fallback:', err)`, and return the original `regexResult` from step 1.

Import GAME_ACTION_SCHEMA from `./ResponseMapper.ts`.

**Preloader modifications** (`src/game/scenes/Preloader.ts`):

1. Import `OllamaClient` from `../llm/OllamaClient`.
2. In `create()`, BEFORE `this.scene.start('MainMenuScene')`, add:
   ```typescript
   // Warm up Ollama LLM model (fire-and-forget, non-blocking)
   // This pre-loads the model into GPU memory so first player command is fast
   const ollamaClient = new OllamaClient();
   ollamaClient.checkAvailability().then(available => {
       if (available) {
           ollamaClient.warmUp(); // Fire-and-forget, catches errors internally
       }
   });
   // Proceed to MainMenu immediately -- warm-up runs in background
   this.scene.start('MainMenuScene');
   ```

The Preloader and HybridParser use separate OllamaClient instances pointing at the same Ollama server. The Preloader's warm-up loads the model into GPU memory; HybridParser's availability check confirms Ollama is running. Both are lightweight operations. OllamaClient has no shared state that matters beyond the `available` boolean cache.

IMPORTANT: The `commandSubmittedHandler` must remain properly typed. Making it async is fine -- the returned promise is ignored by EventBus.

IMPORTANT: Do NOT block the Preloader scene transition on Ollama. The warm-up is fire-and-forget. If Ollama is down, the game proceeds normally with regex-only parsing.
