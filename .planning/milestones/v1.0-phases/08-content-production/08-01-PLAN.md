- Setting, protagonist, central conflict, tone

**Puzzle Dependency Graph (puzzle-dependency-graph.md):**

Using the Ron Gilbert puzzle dependency chart method, create a visual text-based dependency graph for each act. Structure:

For each act:
1. List all puzzles as nodes: `[Puzzle Name] (room_id) -- requires: [items/flags], produces: [items/flags]`
2. Draw ASCII dependency arrows showing prerequisites
3. Mark the act gate (bottleneck between acts)
4. Mark parallel puzzle chains (player can solve in any order)

Format example:
```
=== ACT 1a (Demo) ===

[START: forest_clearing]
    |
    +---> [Get Rusty Key] (forest_clearing) -> flag: has-rusty-key
    |         |
    |         v
    |     [Unlock Cave Door] (cave_entrance) -> flag: door-unlocked
    |
    +---> [Get Stick + Mushroom] (forest/cave) -> item: makeshift-torch
    |
    +---> [Talk to Old Man] (village_path) -> flag: knows_cave_name
              |
              v
          [Learn Cave Secret] (village_path) -> flag: knows_cavern_secret

    ACT GATE: door-unlocked + knows_cavern_secret
```

Validate for EACH puzzle:
- No circular dependencies (A requires B requires A)
- Prerequisites obtainable without solving this puzzle first
- Items consumed only on success
- Act gate verifiable (all required flags/items reachable)

**Room Connection Map (room-map.md):**

Create a text-based map showing all rooms and their connections, organized by act:

```
=== ACT 1a (Demo) ===

forest_clearing <--east/west--> cave_entrance <--east/west--> village_path
                                      |
                                     [door-unlocked]
                                      |
                                      v
                                cave_depths
```

For each room, note:
- Room ID (snake_case, matching story bible)
- Connections (which exits go where, with direction)
- One-way restrictions (if any -- and verify no needed items are behind one-way doors)
- Act boundaries (which rooms connect acts)

Both documents MUST use the exact room IDs, item IDs, flag names, and NPC IDs from the story bible (Task 1). Cross-reference consistency is critical.
