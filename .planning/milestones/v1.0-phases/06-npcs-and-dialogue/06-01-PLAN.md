3. Create `src/game/dialogue/DialogueManager.ts`:
- Import `Story` from `'inkjs'` (NOT from `'inkjs/types'` -- the default export includes the runtime; types-only import may not include the constructor). If the default import does not expose `Story` as a class constructor, try `import { Story } from 'inkjs/engine/Story'` or `const { Story } = await import('inkjs')`. Check that `new Story(jsonString)` works.
- Constructor accepts a `GameState` instance reference.
- `startConversation(npcId: string, compiledJsonString: string): void` -- Creates a new `Story(compiledJsonString)`, restores saved state if `npcStoryStates` has an entry for this npcId, binds all EXTERNAL functions, stores as `activeStory` and `activeNpcId`.
- `continueAll(): { lines: string[]; tags: string[]; choices: Array<{ index: number; text: string }>; ended: boolean }` -- Drains `canContinue` loop collecting all lines and last tags, then reads `currentChoices`. If no canContinue and no choices, `ended = true`.
- `choose(index: number): void` -- Calls `activeStory.ChooseChoiceIndex(index)`.
- `endConversation(): void` -- Saves story state via `activeStory.state.toJson()` into `npcStoryStates` Map keyed by activeNpcId, nulls activeStory.
- `isActive(): boolean` -- Returns true if activeStory is not null.
- `getActiveNpcId(): string | null` -- Returns activeNpcId or null.
- `getDialogueStates(): Record<string, string>` -- Returns all saved states as plain object (for GameState serialization).
- `loadDialogueStates(states: Record<string, string>): void` -- Restores saved states from deserialized GameState.

EXTERNAL function bindings (bind ALL before any Continue call):
- `hasItem(itemId: string) -> boolean`: `state.hasItem(itemId)`
- `hasFlag(flagName: string) -> boolean`: `state.isFlagSet(flagName)`
- `setFlag(flagName: string) -> void`: `state.setFlag(flagName, true)`
- `addItem(itemId: string) -> void`: `state.addItem(itemId)`
- `removeItem(itemId: string) -> void`: `state.removeItem(itemId)`
- `visitedRoom(roomId: string) -> boolean`: `state.getData().visitedRooms.includes(roomId)`
- `getDeathCount() -> number`: `state.getData().deathCount`

CRITICAL: Bind external functions BEFORE any `Continue()` call. The ink runtime errors if it encounters an unbound EXTERNAL.

Store `npcStoryStates` as a `Map<string, string>` internally. The Story constructor expects a JSON string, not a parsed object -- if Phaser cache auto-parses it, the caller must `JSON.stringify()` before passing.

3. Create `src/game/dialogue/DialogueUI.ts`:
- This is a lightweight class that formats dialogue output for the existing NarratorDisplay.
- Constructor accepts a `NarratorDisplay` instance.
- `showDialogueLine(speakerName: string, text: string): void` -- Formats as `"{speakerName}: {text}"` and calls `narratorDisplay.typewrite()` with the formatted string. If text is short (<=50 chars), use `showInstant()` instead (matching the existing 50-char threshold pattern from RoomScene).
- `showChoices(choices: Array<{ index: number; text: string }>): void` -- Formats choices as numbered list `"1. {text}\n2. {text}\n..."` and appends to the narrator display element directly (set textContent on the element after a brief delay or append to existing text). Use `showInstant()` since choices should appear immediately, not typewrite.
- `showDialogueWithChoices(speakerName: string, lines: string[], choices: Array<{ index: number; text: string }>): void` -- Combines lines and choices into one formatted block. Lines first (joined with newline), then blank line, then numbered choices. Uses typewrite for the combined text with onComplete callback to ensure full text is visible.
- `clearDialogue(): void` -- Clears the narrator display via `showInstant('')`.
- `parseTags(tags: string[]): Record<string, string>` -- Parses ink tags in `#key:value` format. Returns key-value map. Bare tags (no colon) get value `'true'`.

This class does NOT create DOM elements -- it reuses the existing NarratorDisplay and its underlying #parser-response element. This follows the established NarratorDisplay enhancement pattern.
