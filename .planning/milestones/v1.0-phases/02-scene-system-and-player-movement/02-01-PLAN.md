**1. Create `src/game/entities/Player.ts`:**

Export a `Player` class with these capabilities:

Constructor `(scene: Phaser.Scene, x: number, y: number)`:
- Create a Phaser sprite at (x, y) using 'player' texture key
- Set depth to 50 (between background layers at 0-10 and future foreground at 90+)
- Create 3 animations using `scene.anims.create()`:
  - `player-idle`: frames 0-3, frameRate 4, repeat -1 (loop)
  - `player-walk`: frames 4-11, frameRate 8, repeat -1 (loop)
  - `player-interact`: frames 12-15, frameRate 6, repeat 0 (play once)
- Play `player-idle` immediately
- Store reference to scene and sprite
- Track `isWalking: boolean = false` and `currentTween: Phaser.Tweens.TweenChain | null = null`

`walkTo(path: Array<{x: number; y: number}>, onComplete?: () => void)`:
- If currentTween exists, call `.stop()` on it and also `this.scene.tweens.killTweensOf(this.sprite)` to prevent tween conflicts
- Play 'player-walk' animation (use `true` second param to avoid restart if already playing)
- Set isWalking = true
- Build tween config array from path waypoints: each tween targets the sprite, moves to point.x/point.y, uses Linear ease, duration = (distance / 120) * 1000 (120 pixels/sec). Each tween's onStart flips sprite direction: `this.sprite.flipX = point.x < this.sprite.x`
- Create tween chain via `this.scene.tweens.chain({ tweens: tweenConfigs, onComplete: () => { this.isWalking = false; this.sprite.play('player-idle'); this.currentTween = null; if (onComplete) onComplete(); } })`
- Store the chain as currentTween

`playInteraction()`:
- Play 'player-interact' animation
- Chain 'player-idle' after it completes using `this.sprite.chain('player-idle')`

`getPosition(): { x: number; y: number }`:
- Return `{ x: this.sprite.x, y: this.sprite.y }`

`getSprite(): Phaser.GameObjects.Sprite`:
- Return the sprite (needed for camera follow)

`stopMovement()`:
- Kill any active tween, stop walking, play idle

`destroy()`:
- Stop any active tween, destroy sprite

**2. Create `src/game/systems/NavigationSystem.ts`:**

Export a `NavigationSystem` class:

Constructor `(walkablePoints: Array<{x: number; y: number}>)`:
- Create a `Phaser.Geom.Polygon` from walkablePoints for containment checks
- Create a `NavMesh` from the navmesh library: `new NavMesh([walkablePoints.map(p => ({x: p.x, y: p.y}))])`
- Import NavMesh from 'navmesh' at top of file

`isPointWalkable(x: number, y: number): boolean`:
- Return `this.walkablePolygon.contains(x, y)` using Phaser.Geom.Polygon.Contains static method

`findPath(from: {x:number;y:number}, to: {x:number;y:number}): Array<{x:number;y:number}> | null`:
- First check if `to` is walkable. If not, return null.
- Call `this.navMesh.findPath(from, to)` and return the result
- NavMesh returns `{x,y}[]` or null

`destroy()`:
- Clean up navmesh reference (set to null)

Note on import: The `navmesh` package exports NavMesh as default export. Use `import NavMesh from 'navmesh'` or check the actual export -- if it uses named export, use `import { NavMesh } from 'navmesh'`. The research shows `import { NavMesh } from 'navmesh'` -- try that first. If TypeScript complains about types, add a type declaration or use `// @ts-ignore` as last resort and note it for cleanup.
