**1. Create `src/game/systems/SceneTransition.ts`:**

Export a `SceneTransition` class (or utility object):

`static fadeToRoom(scene: Phaser.Scene, roomId: string, spawnPoint: {x:number;y:number}, duration: number = 500)`:
- Call `scene.cameras.main.fadeOut(duration, 0, 0, 0)` (fade to black)
- Disable input: `scene.input.enabled = false`
- Listen for FADE_OUT_COMPLETE once: `scene.cameras.main.once(Phaser.Cameras.Scene2D.Events.FADE_OUT_COMPLETE, () => { scene.scene.start('RoomScene', { roomId, spawnPoint }); })`

This is a static helper -- RoomScene manages the `isTransitioning` flag itself.

**2. Create `src/game/scenes/RoomScene.ts`:**

Import: Player, NavigationSystem, SceneTransition, RoomData type, EventBus.

Export class `RoomScene extends Phaser.Scene`:

**Properties:**
- `private roomData!: RoomData`
- `private player!: Player`
- `private navigation!: NavigationSystem`
- `private isTransitioning: boolean = false`
- `private exitZones: Array<{rect: Phaser.Geom.Rectangle; exit: ExitData}> = []`
- `private hotspotZones: Array<{rect: Phaser.Geom.Rectangle; hotspot: HotspotData}> = []`

**constructor():** `super('RoomScene')`

**init(data: {roomId: string; spawnPoint?: {x:number;y:number}}):**
- Load room data from cache: `this.roomData = this.cache.json.get('room-' + data.roomId)`
- If data.spawnPoint provided, override roomData.playerSpawn with it
- Reset isTransitioning = false
- Clear exitZones and hotspotZones arrays

**create():**

1. **Background layers:** Iterate roomData.background.layers, add each as `this.add.image(0, 0, layer.key).setOrigin(0, 0).setScrollFactor(layer.scrollFactor).setDepth(index)` (depth 0-9 for backgrounds)

2. **Navigation:** Create NavigationSystem from roomData.walkableArea: `this.navigation = new NavigationSystem(this.roomData.walkableArea)`

3. **Player:** Create player at spawn point: `this.player = new Player(this, spawnX, spawnY)` where spawnX/spawnY come from the (potentially overridden) playerSpawn

4. **Exit zones:** For each exit in roomData.exits, create a Phaser.Geom.Rectangle from exit.zone and store in exitZones array paired with the exit data

5. **Hotspot zones:** For each hotspot in roomData.hotspots, create a Phaser.Geom.Rectangle from hotspot.zone and store in hotspotZones array paired with the hotspot data. Optionally draw semi-transparent debug rectangles (yellow, alpha 0.15) for hotspots and exit zones during development -- use a `DEBUG` flag (const DEBUG = true at top of file, can be toggled).

6. **Camera setup:**
   - `this.cameras.main.setBounds(0, 0, this.roomData.background.worldWidth, 540)`
   - `this.cameras.main.setRoundPixels(true)`
   - `this.cameras.main.startFollow(this.player.getSprite(), true, 0.1, 0.1)` (smooth follow)

7. **Click handler:** `this.input.on('pointerdown', (pointer: Phaser.Input.Pointer) => { ... })`:
   - If isTransitioning, return early
   - Convert to world coordinates: `const worldPoint = this.cameras.main.getWorldPoint(pointer.x, pointer.y)`
   - Check exits first: loop exitZones, if `Phaser.Geom.Rectangle.Contains(zone.rect, worldPoint.x, worldPoint.y)`, find path from player position to a point just inside the exit zone (use center-x of zone, player's current y clamped to walkable area). If path found, `this.player.walkTo(path, () => this.handleExitReached(zone.exit))`. Return.
   - Check hotspots: loop hotspotZones, if point is in hotspot rect, find path to hotspot.interactionPoint. If path found, `this.player.walkTo(path, () => this.player.playInteraction())`. Return.
   - Default: find path from player position to clicked point. If path found (point is walkable), `this.player.walkTo(path)`.

8. **Fade in:** `this.cameras.main.fadeIn(500, 0, 0, 0)`

9. **EventBus:** Emit 'scene-ready' with room data: `EventBus.emit('scene-ready', { sceneKey: 'RoomScene', roomId: this.roomData.id })`

10. **Shutdown cleanup:** `this.events.on(Phaser.Scenes.Events.SHUTDOWN, () => { this.player.destroy(); this.navigation.destroy(); EventBus.removeAllListeners('scene-ready'); })`

**handleExitReached(exit: ExitData):**
- If isTransitioning, return
- Set isTransitioning = true
- Call `SceneTransition.fadeToRoom(this, exit.targetRoom, exit.spawnPoint)`

**Important implementation notes:**
- Use `Phaser.Geom.Rectangle.Contains()` (static method) for hit testing exit/hotspot zones against world coordinates
- When finding path to an exit, the target point must be INSIDE the walkable area (not inside the exit zone which may be outside the walkable polygon). Use the exit's adjacent walkable-area edge point or clamp the exit zone center to the walkable area bounds. A practical approach: use {x: exit.zone.x (for left exits) or exit.zone.x + exit.zone.width (for right exits), y: playerSpawn.y-ish} clamped into walkable polygon. Simplest: define an `interactionPoint` on each exit in the JSON (similar to hotspots). If not present in JSON, use the center of the exit zone clamped to the walkable area Y range.

Actually, to keep it simpler and consistent: in the click handler, when an exit zone is clicked, walk the player toward the exit. The exit zone overlaps with the edge of the walkable area, so walking to the exit zone center's x at the walkable area's y-center should work. Alternatively, check exit overlap in the `update()` loop: each frame, check if the player's position overlaps any exit zone -- when it does, trigger the transition. This is the approach described in the research.

**Preferred approach (from research):** Do NOT walk to a calculated exit point. Instead:
- Click on exit zone -> walk to the nearest walkable point in that direction (the edge of the walkable area)
- In `update()`, each frame check if player position overlaps any exit zone rect
- If overlap detected and not transitioning, trigger handleExitReached

This is more robust. Implement it this way:

**update() method:**
```typescript
update() {
    if (this.isTransitioning) return;
    const pos = this.player.getPosition();
    for (const zone of this.exitZones) {
        if (Phaser.Geom.Rectangle.Contains(zone.rect, pos.x, pos.y)) {
            this.handleExitReached(zone.exit);
            return;
        }
    }
}
```

And in the click handler, when clicking an exit zone area, just walk to the clicked point (it should be walkable if the exit zone overlaps the walkable area edge). If it's not walkable, walk to the nearest walkable point in that direction. Simplest: just walk to the clicked world point if walkable, and let the update() overlap detection handle the transition.
