  <action>
    **Part A: Generate placeholder parallax background images**

    Create 4 placeholder pixel art background images programmatically using a Node.js script (use the `canvas` npm package, or write a minimal script that creates PNG files). Each image should be 1920x540 pixels (2x the game width of 960, giving room to scroll). The images should be simple but visually distinct so parallax depth is obvious:

    - `sky.png` -- Solid gradient from dark blue (#0a0a2e) at top to lighter blue (#1a1a4e) at bottom. This is the fixed background.
    - `mountains.png` -- Dark silhouette mountain shapes against transparent background. Simple triangular peaks. Use dark purple/gray (#2a1a3e) with alpha transparency above the mountain line.
    - `trees.png` -- Medium-height tree silhouettes against transparent background. Use dark green (#1a3a1a) with alpha transparency above the treeline.
    - `ground.png` -- Ground/terrain strip at the bottom ~150px. Use brown (#3a2a1a) with some pixel variation for texture. Transparent above.

    If the `canvas` npm package is too heavy, alternatively create these as simple procedural textures using a Python script (if python3 + PIL available), or use any tool available on the system. The images are placeholders -- they just need to be visually distinct colored layers so parallax is visible.

    Important: These images should NOT be power-of-two constrained since we are using `setScrollFactor()` on regular Image objects (not TileSprites). The POT requirement only applies to TileSprites.

    **Part B: Update Preloader to load parallax assets**

    In `src/game/scenes/Preloader.ts`, add load calls in `preload()`:
    ```typescript
    this.load.image('bg-sky', 'assets/backgrounds/sky.png');
    this.load.image('bg-mountains', 'assets/backgrounds/mountains.png');
    this.load.image('bg-trees', 'assets/backgrounds/trees.png');
    this.load.image('bg-ground', 'assets/backgrounds/ground.png');
    ```

    **Part C: Implement parallax scrolling in Game scene**

    Replace the placeholder Game scene content with full parallax implementation:

    In `create()`:
    1. Define `worldWidth = 1920` (matches background image width)
    2. Add layers back-to-front with different scroll factors:
       - Sky: `this.add.image(0, 0, 'bg-sky').setOrigin(0, 0).setScrollFactor(0)` -- fixed background, never moves
       - Mountains: `this.add.image(0, 0, 'bg-mountains').setOrigin(0, 0).setScrollFactor(0.1)` -- very slow parallax
       - Trees: `this.add.image(0, 0, 'bg-trees').setOrigin(0, 0).setScrollFactor(0.4)` -- medium parallax
       - Ground: `this.add.image(0, 0, 'bg-ground').setOrigin(0, 0).setScrollFactor(1)` -- moves with camera
    3. Set camera bounds: `this.cameras.main.setBounds(0, 0, worldWidth, 540)`
    4. Set round pixels: `this.cameras.main.setRoundPixels(true)`
    5. Add keyboard controls for testing parallax:
       - Create cursor keys: `this.cursors = this.input.keyboard!.createCursorKeys()`
       - Store as class property with type `Phaser.Types.Input.Keyboard.CursorKeys`
    6. Add a small info text in the top-left (fixed to camera) showing "Arrow keys to scroll | Phase 1 Foundation" using `setScrollFactor(0)` so it stays on screen
    7. Emit `EventBus.emit('scene-ready', { sceneKey: 'Game' })` and clean up on SHUTDOWN

    In `update(time: number, delta: number)`:
    1. Read cursor key state:
       - If right arrow held: `this.cameras.main.scrollX += 3` (3 pixels per frame at 60fps = smooth scroll)
       - If left arrow held: `this.cameras.main.scrollX -= 3`
    2. Clamp camera within bounds (camera.setBounds already does this, but verify)

    The scroll factor values (0, 0.1, 0.4, 1) create the parallax depth:
    - Sky at 0 stays fixed (infinitely far away)
    - Mountains at 0.1 move at 10% of camera speed (very distant)
    - Trees at 0.4 move at 40% of camera speed (mid-distance)
    - Ground at 1.0 moves with the camera (foreground)
  </action>
