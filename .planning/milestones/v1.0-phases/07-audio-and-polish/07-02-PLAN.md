  <action>
Integrate AudioManager into RoomScene so audio plays during gameplay.

**1. Import AudioManager** at the top of RoomScene.ts:
```typescript
import { AudioManager } from '../systems/AudioManager';
```

**2. Add AudioManager instance variable** in the class properties section (after the Phase 6 dialogue variables):
```typescript
// Phase 7 audio integration
private audioManager!: AudioManager;
```

**3. Initialize AudioManager in `create()` method** -- add after the DialogueManager initialization (after the "Restore dialogue states" block, around line 324), BEFORE the EventBus command-submitted listener setup:
```typescript
// 8a. AudioManager initialization
this.audioManager = AudioManager.getInstance();
this.audioManager.init(this);
this.audioManager.onRoomEnter(this.roomData);
```

This must come BEFORE the EventBus listeners because AudioManager.init() registers its own EventBus listeners for SFX, and we want SFX to work from the first command.

The `onRoomEnter(this.roomData)` call reads `this.roomData.audio` to start the room's music and ambient tracks. If a room has no `audio` field, it gracefully does nothing for music and clears ambient (since no ambient is specified).

**4. Add AudioManager cleanup to the SHUTDOWN handler** -- in the existing `this.events.on(Phaser.Scenes.Events.SHUTDOWN, () => { ... })` block, add before the Phase 6 dialogue cleanup:
```typescript
// Phase 7 audio cleanup (remove EventBus listeners, keep audio playing for crossfade)
this.audioManager.cleanup();
```

The cleanup() method removes EventBus listeners but does NOT stop audio. This is intentional: music continues playing during the scene transition and the new scene's AudioManager.init() + onRoomEnter() will crossfade to the new track (or keep playing if same music key).

**5. No changes to the event handler functions needed** -- AudioManager listens to the same EventBus events (`item-picked-up`, `go-command`, `trigger-death`, `start-dialogue`, `command-submitted`) that are already emitted by the existing systems. SFX plays automatically via those listeners.

**Important:** Do NOT add any `this.sound.play()` calls directly in RoomScene. All audio goes through AudioManager. Do NOT call `this.audioManager.stopAll()` on shutdown -- music should persist across scene transitions for crossfading.
