Create `src/game/ui/NarratorDisplay.ts`:
A class that wraps an HTMLElement and provides typewriter text reveal via setInterval.

Constructor takes an HTMLElement (the existing `#parser-response` element from TextInputBar, or a new element). Store reference.

Methods:
- `typewrite(text: string, speed: number = 30, onComplete?: () => void)`: Stop any existing timer. Set fullText and charIndex=0. Clear element textContent. Start setInterval that appends one character per tick. When charIndex reaches fullText.length, stop timer and call onComplete.
- `skipToEnd()`: Stop timer, set element.textContent to fullText, call onComplete.
- `showInstant(text: string)`: Stop timer, set fullText and textContent immediately.
- `stop()` (private): clearInterval if timer exists.
- `destroy()`: Call stop().

Add click-to-skip: In constructor, add click event listener on element that calls skipToEnd(). Store the listener reference for cleanup in destroy().

Also add keyboard skip: add a keydown listener on document that calls skipToEnd() when typewriter is active (timer is not null). Remove in destroy(). Only skip on printable keys, Enter, or Space -- NOT arrow keys (those navigate command history).

CSS: The NarratorDisplay does not create its own DOM element -- it enhances an existing element. The typewriter effect works on whatever element is passed to the constructor.

Create `src/game/ui/InventoryPanel.ts`:
An HTML panel that displays inventory items. Follows the same pattern as TextInputBar (HTML overlay appended to game-container).

Constructor takes container HTMLElement. Creates:
- `panelEl` (div#inventory-panel): The wrapper. `display: none` by default.
- `headerEl` (div#inventory-header): Text "Inventory"
- `itemsEl` (div#inventory-items): Container for item elements
- `closeBtn` (span.inventory-close): "X" button that calls hide()

Apply inline styles matching the dark theme of the game:
- Panel: position absolute, background #1a1a2e with slight transparency (rgba(26,26,46,0.95)), border 1px solid #444, padding 12px, width matching canvas width, placed above the text input bar area.
- Header: color #c8c8d4, font monospace 14px bold, margin-bottom 8px.
- Items container: display flex, flex-wrap wrap, gap 8px.
- Each item: background #2a2a3e, border 1px solid #555, padding 6px 10px, color #c8c8d4, font monospace 12px, cursor pointer.
- Empty state: italic text "Your pockets are empty. How tragic." in #888.
- Close button: position absolute top-right, cursor pointer, color #888, hover #fff.

Methods:
- `update(items: Array<{ id: string; name: string }>)`: Clear itemsEl innerHTML. If empty, show empty state text. Otherwise create a div.inventory-item for each item with textContent = item.name and data-item-id = item.id.
- `toggle()`: Flip visible state, set display accordingly.
- `show()`: Set visible=true, display='block'.
- `hide()`: Set visible=false, display='none'.
- `isVisible()`: Return visible state.
- `destroy()`: Remove panelEl from DOM.

**Extend `src/game/types/GameAction.ts`:**
Add 'inventory', 'save', 'load', 'combine' to the Verb union type:
```typescript
export type Verb = 'look' | 'take' | 'use' | 'go' | 'talk' | 'open' | 'push' | 'pull'
    | 'inventory' | 'save' | 'load' | 'combine';
```
Keep all existing interfaces (GameAction, ParseResult) unchanged.

**Extend `src/game/parser/VerbTable.ts`:**
Add four new entries to the VERB_TABLE array:

1. `inventory`: canonical 'inventory', synonyms ['inventory', 'i', 'items', 'bag', 'pockets'], patterns: `/^(?:inventory|items|bag|pockets|i)$/i` (bare command only, no noun).

2. `combine`: canonical 'combine', synonyms ['combine', 'merge', 'mix'], patterns: `/^(?:combine|merge|mix)\s+(.+?)\s+(?:and|with)\s+(.+)$/i` (two-noun: "combine X with Y"). Note: "combine" already exists as a synonym of "use" in the current VerbTable. REMOVE 'combine' from the 'use' verb's synonyms list to avoid conflict. The "use X with Y" pattern still handles "use X with Y" via the 'use' verb. "combine X with Y" now routes to the dedicated 'combine' verb.

3. `save`: canonical 'save', synonyms ['save', 'savegame'], patterns: `/^save\s+(?:game\s+)?(\d+)$/i` (save to slot number), `/^save(?:\s+game)?$/i` (bare save -- prompts for slot).

4. `load`: canonical 'load', synonyms ['load', 'restore', 'loadgame'], patterns: `/^(?:load|restore)\s+(?:game\s+)?(\d+)$/i` (load from slot), `/^(?:load|restore)(?:\s+game)?$/i` (bare load -- shows slots).

Place these AFTER the existing 8 verb entries (they are lower priority than adventure verbs).

**Extend `src/game/parser/NounResolver.ts`:**
1. Add 'item' to the ResolvedNoun type union: `type: 'hotspot' | 'exit' | 'direction' | 'item' | 'unknown'`

2. Add optional `inventoryItems` parameter to the `resolve()` method signature:
```typescript
resolve(
    rawNoun: string,
    hotspots: HotspotData[],
    exits: ExitData[],
    inventoryItems?: Array<{ id: string; name: string }>,
): ResolvedNoun
```

3. After step 3 (partial hotspot match) and BEFORE step 4 (direction mapping), add inventory item resolution:
   - Exact ID match: `if (item.id === normalized)` -> return { type: 'item', id: item.id, confidence: 'exact' }
   - Exact name match: `if (item.name.toLowerCase() === normalized)` -> return { type: 'item', id: item.id, confidence: 'exact' }
   - Partial word match: check if any word in normalized matches any word in item.name -> return { type: 'item', id: item.id, confidence: 'partial' }

This ensures inventory items are checked after room hotspots (room objects take visual priority) but before directions/exits (items are more specific than navigation). This matches the research pitfall #2 guidance about extending NounResolver for inventory.

4. Do NOT change the existing TextParser.ts call to NounResolver -- it currently doesn't pass inventoryItems, so the new parameter defaults to undefined and the new code path is skipped. The CommandDispatcher (Plan 03) will be the caller that passes inventory items.

Run existing parser tests to confirm nothing breaks: `npx vitest run src/game/parser/__tests__/TextParser.test.ts`
