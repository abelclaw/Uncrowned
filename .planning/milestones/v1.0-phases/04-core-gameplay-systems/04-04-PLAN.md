  <action>
Create `src/game/scenes/DeathScene.ts` -- a Phaser.Scene with key 'DeathScene'.

This scene is launched as an OVERLAY via `scene.launch('DeathScene', data)` (not scene.start), so the RoomScene remains running (but paused) underneath.

**create(data) receives:**
```typescript
{ title: string; narratorText: string }
```

**Layout (960x540 game resolution):**
1. Dark overlay covering entire viewport: `this.add.rectangle(480, 270, 960, 540, 0x000000, 0.85).setDepth(0)`. Make it interactive to capture clicks and prevent pass-through to RoomScene: `.setInteractive()`.

2. Death title (e.g., "Death by Curiosity"): Centered, y ~130. Font: monospace, 28px, color #cc3333. setOrigin(0.5).

3. Death count: Small text below title showing "Deaths: {N}". Read from GameState.getInstance().getData().deathCount. Font: monospace, 12px, color #666. y ~165.

4. Narrator text: Centered, y ~280. Font: monospace, 16px, color #c8c8d4, wordWrap width 700, align center, lineSpacing 8. setOrigin(0.5). Display the full narratorText. (Typewriter effect on Phaser text is complex and not worth it here -- the HTML NarratorDisplay handles typewriter for in-game text. Death screen text appears instantly.)

5. "[ Try Again ]" button: Centered, y ~430. Font: monospace, 22px, color #ffcc00. setInteractive with useHandCursor.
   - pointerover: setColor('#ffffff'), setScale(1.05)
   - pointerout: setColor('#ffcc00'), setScale(1.0)
   - pointerdown: Execute retry logic

**Retry logic (on Try Again click):**
1. Increment deathCount: `GameState.getInstance().getData().deathCount++` (direct mutation is ok since GameState data is the singleton's internal state; but better to add an `incrementDeathCount()` method -- if it doesn't exist, just do the increment directly on the data).
2. Load auto-save: `SaveManager.loadAutoSave(GameState.getInstance())`
3. Get restored room ID: `GameState.getInstance().getData().currentRoom`
4. Stop the DeathScene: `this.scene.stop('DeathScene')`
5. Restart RoomScene with the restored room: `this.scene.start('RoomScene', { roomId })`

Note: `scene.start` on RoomScene will automatically stop the current RoomScene instance and start fresh, which is the desired behavior (clean room reload from auto-save state).

**Prevent input leak:**
The overlay rectangle's setInteractive() captures clicks. Additionally, the scene is launched while RoomScene remains visible but should be paused. The caller (RoomScene in Plan 05) will handle pausing itself before launching DeathScene.

**Import:** GameState from state/GameState, SaveManager from state/SaveManager.

**DEATH-05 (unwinnable state prevention):** This is primarily a design constraint enforced by:
- Auto-save on room entry (items are only consumed by successful puzzle actions)
- Death resets to room-entry state (no mid-room progress lost)
- Items never consumed on failed attempts (enforced in CommandDispatcher, Plan 03)
The DeathScene's retry mechanism is the runtime enforcement: death always restores to a known-good state.
