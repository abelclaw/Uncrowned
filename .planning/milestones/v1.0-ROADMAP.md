## Overview

Complete King's Quest-inspired browser adventure game engine with a 5-hour fantasy adventure. Features pixel art rendering, point-and-click movement, natural language text parser (deterministic + LLM), inventory/puzzle/death systems, NPC dialogue via inkjs, audio, and 36 rooms of authored content across 4 acts.

## Phases

### Phase 1: Foundation and Rendering

**Goal**: A pixel-perfect HTML5 Canvas renders scenes and runs a stable game loop that all future systems build on
**Depends on**: Nothing (first phase)
**Plans**: 2 plans

Plans:
- [x] 01-01: Project scaffolding, pixel-perfect game config, Boot/Preloader/Game scene chain, EventBus
- [x] 01-02: Parallax scrolling backgrounds, placeholder art assets, visual verification

**Details:**
960x540 game resolution (2x of 480x270 native pixel art). Phaser 3 with Vite bundler. Graphics primitives for boot progress bar. Parallax scroll factors 0/0.1/0.4/1.0 for sky/mountains/trees/ground. 1920x540 background images for horizontal scroll.

### Phase 2: Scene System and Player Movement

**Goal**: Player navigates between interconnected rooms with an animated character that walks to clicked locations
**Depends on**: Phase 1
**Plans**: 2 plans

Plans:
- [x] 02-01: Player entity with animations, NavigationSystem with navmesh pathfinding, room data types, placeholder assets
- [x] 02-02: RoomScene with click-to-move pipeline, SceneTransition with fade effects, multi-room navigation

**Details:**
Standalone navmesh library (not phaser-navmesh plugin). Convex rectangular walkable areas. Player sprite depth 50. Exit overlap detection in update() loop. SceneTransition supports fade and slide. Hotspots checked before exits in click handler.

### Phase 3: Text Parser (Deterministic)

**Goal**: Player types text commands and the game reliably understands standard adventure game verbs without any LLM dependency
**Depends on**: Phase 2
**Plans**: 2 plans

Plans:
- [x] 03-01: TDD: GameAction types, TextParser with VerbTable and NounResolver, test suite
- [x] 03-02: TextInputBar UI, CommandDispatcher, room JSON text content, RoomScene wiring

**Details:**
TDD approach with Vitest. 8 standard verbs (look, take, use, go, talk, open, push, pull). VerbTable with 12 entries and 40+ synonyms. NounResolver with 6-strategy fuzzy matching. TextInputBar as HTML overlay below canvas. CommandDispatcher with double-resolution (ID, name, partial). 50 tests covering all verbs.

### Phase 4: Core Gameplay Systems

**Goal**: The complete adventure game loop works: player picks up items, solves puzzles, dies humorously, saves progress, and hears the narrator's sardonic voice
**Depends on**: Phase 3
**Plans**: 5 plans

Plans:
- [x] 04-01: TDD: GameState singleton, PuzzleEngine condition/action evaluator, SaveManager, type definitions
- [x] 04-02: NarratorDisplay typewriter effect, InventoryPanel HTML UI, parser verb/noun extensions
- [x] 04-03: CommandDispatcher overhaul with PuzzleEngine integration, enriched room JSON data
- [x] 04-04: MainMenuScene and DeathScene overlay Phaser scenes
- [x] 04-05: Full RoomScene integration, auto-save wiring, gameplay loop verification

**Details:**
GameState singleton with inventory, flags, rooms, serialization. PuzzleEngine with JSON-defined condition/action pairs. SaveManager with auto-save + 5 manual slots via localStorage. NarratorDisplay typewriter threshold at 50 chars. InventoryPanel HTML overlay. DeathScene as overlay via scene.launch. Items consumed only by explicit puzzle actions (prevents unwinnable states). Combine handler checks both orderings. Auto-save records destination room.

### Phase 5: LLM Integration

**Goal**: Natural language commands like "maybe I should pick up that shiny thing" work seamlessly, making the text parser feel magical
**Depends on**: Phase 4
**Plans**: 2 plans

Plans:
- [x] 05-01: OllamaClient fetch wrapper, PromptBuilder context assembly, ResponseMapper with NounResolver
- [x] 05-02: HybridParser orchestrator, RoomScene async command wiring, Preloader model warm-up

**Details:**
Raw fetch over ollama npm package (single endpoint doesn't justify 15KB dependency). AbortSignal.any combines timeout + manual abort. GAME_ACTION_SCHEMA excludes save/load meta-commands. NounResolver reused from parser module. Thinking indicator ('...') shown via showInstant before async parse. Post-await isTransitioning re-check prevents stale action dispatch.

### Phase 6: NPCs and Dialogue

**Goal**: NPCs populate the world with personalities, branching conversations, and puzzle-relevant information driven by inkjs narrative scripting
**Depends on**: Phase 5
**Plans**: 3 plans

Plans:
- [x] 06-01: Dialogue engine foundation: inkjs, NPC types, DialogueManager, DialogueUI, GameState extension
- [x] 06-02: NPC content: npcs.json registry, ink dialogue scripts, room JSON NPC entries, Preloader loading
- [x] 06-03: Command pipeline integration: CommandDispatcher NPC detection, RoomScene dialogue mode

**Details:**
Named import { Story } from 'inkjs'. Map<string,string> for runtime npcStoryStates, Record for serialization. DialogueUI reuses NarratorDisplay with 50-char typewrite threshold. inkjs Compiler class for pre-compilation. Dialogue mode input routing before HybridParser.parse(). NPC zones rendered as synthetic hotspots. Narrator_history runs as non-interactive one-shot conversation.

### Phase 7: Audio and Polish

**Goal**: The game world feels alive with sound and the experience is polished for players
**Depends on**: Phase 4
**Plans**: 2 plans

Plans:
- [x] 07-01: AudioManager singleton, RoomData audio types, audio registry, placeholder audio files
- [x] 07-02: Preloader audio loading, room JSON audio fields, RoomScene AudioManager integration

**Details:**
AudioManager singleton wraps Phaser global Sound Manager. Three layers: SFX, music, ambient. EventBus-driven SFX via audio-registry.json. WAV format for placeholder audio. Python wave module for generation. cleanup() on shutdown removes listeners but preserves audio for crossfade.

### Phase 8: Content Production

**Goal**: A complete 5-hour fantasy adventure authored and playtested (art generation via Flux deferred to v2)
**Depends on**: Phases 6, 7
**Plans**: 4 plans

Plans:
- [x] 08-01: Story bible, puzzle dependency graph, and room connection map
- [x] 08-02: Demo chapter (Act 1a): room JSONs, items, NPCs, ink dialogue
- [x] 08-03: Acts 1b and 2: room JSONs, items, NPCs, ink dialogue
- [x] 08-04: Act 3 (climax/resolution): room JSONs, items, NPCs, full game validation

**Details:**
Kingdom of Erelhain setting with petrification curse. Protagonist Pip -- reluctant tinker mistaken for prophesied savior. The Clerk as immortal bureaucrat antagonist/redeemable. Rite of Administrative Closure as climax (bureaucratic form-filing breaks the curse). 4-act structure: Demo (7 rooms), Royal Mess (7), Screaming Caverns (12), Rite (10) = 36 rooms. Alternative puzzle solutions where reasonable. 597-line story bible, 490-line puzzle dependency graph, 222-line room map. 37 items, 11 NPCs, 12 ink dialogue scripts.
