**In src/game/types/RoomData.ts:**

1. Add a `PostFXType` string literal union: `'bloom' | 'glow' | 'desaturate'`

2. Add a `LightingData` interface:
```typescript
export interface LightingData {
  tint?: string;           // hex color string e.g. "#ff8844" applied as camera tint
  brightness?: number;     // 0.0-1.0, default 1.0 (1.0 = normal, 0.3 = very dark)
  vignette?: number;       // 0.0-1.0 vignette intensity, 0 = none, 0.8 = heavy darkened edges
  torchFlicker?: boolean;  // if true, brightness oscillates to simulate torch light
  postfx?: PostFXType[];   // list of PostFX effects to apply
}
```

3. Add `lighting?: LightingData` to the existing `RoomEffectsData` interface (alongside weather and ambient).

**In src/game/systems/EffectsManager.ts:**

1. Import `LightingData` from types.

2. Add private fields:
   - `private lightingOverlay: Phaser.GameObjects.Rectangle | null = null;` (for tint/brightness)
   - `private vignetteGraphics: Phaser.GameObjects.Graphics | null = null;` (for vignette)
   - `private flickerTimer: Phaser.Time.TimerEvent | null = null;` (for torch flicker)
   - `private postFXPipeline: Phaser.FX.Controller[] = [];` (track applied PostFX for cleanup)

3. Add `private applyLighting(config: LightingData): void` method:
   - **Tint + Brightness**: Create a full-screen `Rectangle(0, 0, GAME_WIDTH, GAME_HEIGHT)` at depth 70 (below ambient 75), setScrollFactor(0), setOrigin(0). Parse the tint hex string to a number (e.g. `parseInt(config.tint.replace('#',''), 16)`). Set the rectangle fill color to the tint color. Set alpha based on `1 - (config.brightness ?? 1.0)` -- lower brightness = higher overlay alpha. If no tint specified, use black (0x000000) for pure darkness. Store as `this.lightingOverlay`.
   - **Vignette**: If `config.vignette` > 0, use `this.scene.cameras.main.postFX.addVignette(0.5, 0.5, config.vignette)`. Store the returned controller in `this.postFXPipeline`.
   - **Torch Flicker**: If `config.torchFlicker` is true, create a Phaser TimerEvent that fires every 100-200ms (random interval). On each tick, slightly randomize the lightingOverlay alpha around the base brightness value (+/- 0.05-0.1). This creates a subtle flickering torch effect. Store as `this.flickerTimer`. Use `this.scene.time.addEvent({ delay: 150, callback: ..., loop: true })`.
   - **PostFX**: For each entry in `config.postfx`:
     - `'bloom'`: `this.scene.cameras.main.postFX.addBloom(0xffffff, 1, 1, 1, 1.2)` -- subtle warm bloom
     - `'glow'`: `this.scene.cameras.main.postFX.addGlow(0xffff88, 2, 0, false, 0.1, 10)` -- soft glow overlay
     - `'desaturate'`: `this.scene.cameras.main.postFX.addColorMatrix().desaturate()` -- drain color for eerie mood
     Store each returned controller in `this.postFXPipeline` array.

4. Add `private clearLighting(): void` method:
   - Destroy `lightingOverlay` if exists, null it
   - Destroy `vignetteGraphics` if exists, null it
   - Stop and remove `flickerTimer` if exists, null it
   - Remove all PostFX: call `this.scene.cameras.main.postFX.clear()` (removes all PostFX from camera). Empty `this.postFXPipeline` array.

5. Update `onRoomEnter()`: After existing weather/ambient handling, call `this.clearLighting()` then check `roomData.effects?.lighting` and call `this.applyLighting()` if present.

6. Update `clearAll()`: Call `this.clearLighting()` alongside existing weather/ambient cleanup.

**Depth layering**: Lighting overlay at depth 70 (below ambient 75, weather 80, sparkle 85). This means particles float ABOVE the tint overlay, which looks correct -- rain and fireflies should be visible even in dark rooms.
