3. Create the compile script `scripts/compile-ink.mjs`:
```javascript
import { Compiler } from 'inkjs/compiler/Compiler';
import { readFileSync, writeFileSync, readdirSync, mkdirSync, existsSync } from 'fs';
import { join, basename } from 'path';

const sourceDir = 'public/assets/data/ink-source';
const outDir = 'public/assets/data/dialogue';

if (!existsSync(outDir)) mkdirSync(outDir, { recursive: true });

const inkFiles = readdirSync(sourceDir).filter(f => f.endsWith('.ink'));

for (const file of inkFiles) {
  const source = readFileSync(join(sourceDir, file), 'utf-8');
  const compiler = new Compiler(source);
  const story = compiler.Compile();
  const json = story.ToJson();
  const outName = basename(file, '.ink') + '.ink.json';
  writeFileSync(join(outDir, outName), json);
  console.log(`Compiled ${file} -> ${outName}`);
}
```

Run `node scripts/compile-ink.mjs` to generate the compiled JSON files. If the `inkjs/compiler/Compiler` import path doesn't work, try `import { InkCompiler as Compiler } from 'inkjs/full'` or `import inkjs from 'inkjs/full'` and use `new inkjs.Compiler(source)`. The inkjs package exposes the Compiler through the `inkjs/compiler/Compiler` path or through the full bundle.

FALLBACK: If the Compiler class is not accessible from the npm inkjs package (some versions bundle it differently), use the Inky desktop app or inklecate CLI to compile. As a last resort, use runtime compilation by importing from `inkjs/full` in the DialogueManager instead of pre-compiling. In that case, store raw .ink text as .json strings and compile on first load.
