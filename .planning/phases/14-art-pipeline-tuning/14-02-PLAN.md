<objective>
Add a LoRA strength test matrix CLI mode and calibrate values through human visual review.

Purpose: The starting LoRA strength estimates (bg: 0.7, sprites: 0.85, items: 0.9) are educated guesses. Different rooms across forest, cave, and twilight environments respond differently to LoRA intensity. A structured test matrix generates 27 background comparisons (3 rooms x 3 acts x 3 strengths) that the user reviews to select optimal values, which then get written back to style-guide.json for Phase 15 batch generation.

Output: `--test-lora` CLI flag in generate-art.ts, test output images in test-output/, and validated LoRA strength values in style-guide.json.
</objective>

  <name>Task 1: Add --test-lora CLI mode for LoRA strength comparison grid</name>
  <files>scripts/generate-art.ts</files>
  <action>
Add a `--test-lora` flag to the CLI argument parser and implement a test matrix generation mode.

1. Add `testLora: boolean` to the `CLIArgs` interface (default false).

2. Add `'--test-lora'` case to the `parseArgs()` switch statement.

3. Update `printHelp()` to document the new flag:
   ```
   --test-lora      Generate LoRA strength comparison grid (9 rooms x 3 strengths = 27 images)
   ```

4. Define the test matrix constants (at module level or in a dedicated function):
```typescript
const TEST_ROOMS: Record<string, string[]> = {
    act1: ['forest_clearing', 'throne_room', 'castle_garden'],
    act2: ['crystal_chamber', 'cavern_library', 'forge_chamber'],
    act3: ['petrified_forest', 'wizard_tower', 'rooftop'],
};

const LORA_STRENGTHS_TO_TEST = [0.6, 0.7, 0.8];
```

5. Add a `runTestLoraMatrix()` async function that:
   a. Checks ComfyUI is running (reuse `checkComfyUI()`)
   b. Creates output directory: `test-output/lora-comparison/`
   c. For each act, for each room in that act's test rooms:
      - Look up the room's manifest entry from `manifest.roomBackgrounds[roomId]`
      - For each LoRA strength in LORA_STRENGTHS_TO_TEST:
        - Create an AssetTypeConfig with the background resolution (1024x576) but the test LoRA strength
        - Build the full prompt (same as normal background generation using styleGuide.promptPrefix etc.)
        - Call `generateImage()` with the overridden config
        - Post-process with `processBackground()` to 960x540
        - Save to `test-output/lora-comparison/{act}/{roomId}-lora-{strength}.png`
        - Log progress: `[N/27] {roomId} @ LoRA {strength} (Xs)`
   d. Print summary when done: total generated, output directory path
   e. Print review instructions: "Review images in test-output/lora-comparison/ -- compare LoRA strengths per room and per act. Pick the strength that best balances pixel art style with scene composition."

6. In `main()`, add an early branch: if `args.testLora`, call `runTestLoraMatrix()` and return (skip normal generation flow).

7. Add `test-output/` to `.gitignore` if not already present (check first).

**Dry-run support:** If `--dry-run` is also passed with `--test-lora`, list the 27 planned generations without running them. This lets the user verify the test matrix before committing GPU time.
