<task type="auto">
  <name>Task 1: Viewport meta tag, dvh layout, touch-action, and mobile input font size</name>
  <files>index.html, public/style.css</files>
  <action>
1. In index.html, replace the existing viewport meta tag:
   FROM: `<meta name="viewport" content="width=device-width, initial-scale=1.0" />`
   TO: `<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no, viewport-fit=cover" />`

2. In public/style.css, update the `#app` rule to use dvh with vh fallback:
   - Keep `height: 100vh;` as first declaration (fallback for older browsers)
   - Add `height: 100dvh;` as second declaration (overrides in modern browsers)
   - The existing `overflow: hidden`, `display: flex`, etc. stay unchanged

3. In public/style.css, add safe-area padding to `#game-container`:
   - Add `padding-bottom: env(safe-area-inset-bottom, 0px);` for notched iPhones (required with viewport-fit=cover)

4. In public/style.css, add `touch-action: none;` rule to `#game-container canvas` (append to existing rule block that has image-rendering). This prevents browser scroll/zoom gestures on the canvas and lets Phaser receive all touch events.

5. In public/style.css, add a mobile media query block at the end:
   ```css
   @media (max-width: 768px) {
       #parser-input {
           font-size: 16px;
       }
   }
   ```
   This prevents iOS Safari auto-zoom when the user taps the text input (iOS zooms if input font-size < 16px).

Do NOT change any existing desktop styles. All mobile additions are additive (dvh overrides vh in supporting browsers, media query only applies on narrow viewports).
  </action>
  <verify>
Run `npx vite build` to confirm no build errors. Visually inspect index.html for the new meta tag. Grep style.css for `100dvh`, `touch-action: none`, `font-size: 16px`, and `safe-area-inset-bottom` to confirm all rules are present.
  </verify>
  <done>
index.html has the MOBI-06 viewport meta tag. style.css has dvh layout (MOBI-01), touch-action on canvas, safe-area padding, and 16px mobile input font (MOBI-05). No desktop styles are broken.
  </done>

<task type="auto">
  <name>Task 2: Create VerbBar HTML component with EventBus integration</name>
  <files>src/game/ui/VerbBar.ts, public/style.css</files>
  <action>
1. Create src/game/ui/VerbBar.ts following the exact same pattern as TextInputBar.ts (HTML element creation, appended to a container element, EventBus integration):

   - Import EventBus from '../EventBus'
   - Define a const VERBS array: [{ label: 'Look', command: 'look' }, { label: 'Take', command: 'take' }, { label: 'Use', command: 'use' }, { label: 'Go', command: 'go' }, { label: 'Talk', command: 'talk' }, { label: 'Inventory', command: 'inventory' }]
   - Export class VerbBar with:
     - private wrapperEl: HTMLDivElement
     - constructor(container: HTMLElement): Creates a div with id='verb-bar', creates 6 button elements with class='verb-btn', each button's click handler calls EventBus.emit('command-submitted', verb.command). Set type='button' attribute on each button to prevent form submission. Append wrapperEl to container.
     - show(): void -- sets wrapperEl.style.display = 'flex'
     - hide(): void -- sets wrapperEl.style.display = 'none'
     - destroy(): void -- calls wrapperEl.remove()

   The VerbBar emits the EXACT same 'command-submitted' event that TextInputBar uses, so the existing RoomScene command handler will process verb button taps identically to typed commands. For verbs that need a target (like "look" or "take"), the RoomScene's existing CommandDispatcher will respond with the appropriate prompt (e.g., "Look at what?").

2. In public/style.css, add the VerbBar styles AFTER the existing parser styles:

   ```css

   #verb-bar {
       display: none;
       width: 100%;
       max-width: 960px;
       flex-direction: row;
       flex-wrap: nowrap;
       background: #1a1a2e;
       border-top: 1px solid #16213e;
       box-sizing: border-box;
       padding-bottom: env(safe-area-inset-bottom, 0px);
   }

   .verb-btn {
       flex: 1;
       min-height: 44px;
       background: #2a2a3e;
       border: none;
       border-right: 1px solid #16213e;
       color: #c8c8d4;
       font-family: 'Courier New', Courier, monospace;
       font-size: 12px;
       cursor: pointer;
       padding: 8px 4px;
       touch-action: manipulation;
       -webkit-tap-highlight-color: transparent;
   }

   .verb-btn:last-child {
       border-right: none;
   }

   .verb-btn:active {
       background: #3a3a5e;
   }
   ```

   Then update the mobile media query (from Task 1) to also show the verb bar:
   ```css
   @media (max-width: 768px) {
       #parser-input {
           font-size: 16px;
       }
       #verb-bar {
           display: flex;
       }
   }
   ```

   The verb bar is `display: none` by default (desktop), and `display: flex` inside the 768px media query (mobile). This satisfies MOBI-02: buttons visible on mobile, hidden on desktop.
