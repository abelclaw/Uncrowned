<task type="auto">
  <name>Task 1: Create MobileKeyboardManager with visualViewport listener and pinch-zoom prevention</name>
  <files>src/game/systems/MobileKeyboardManager.ts, src/game/main.ts</files>
  <action>
1. Create src/game/systems/MobileKeyboardManager.ts:

   Export a class MobileKeyboardManager with:
   - private appEl: HTMLElement (the #app element)
   - private lockedHeight: number | null = null

   constructor(appEl: HTMLElement):
   - Store appEl reference
   - If window.visualViewport exists, add a 'resize' event listener (this.onVisualViewportResize, bound as arrow function)
   - Call this.preventPinchZoom() to set up iOS gesture prevention

   private onVisualViewportResize = (): void:
   - Get visualViewport height (vv.height)
   - Determine if keyboard is open: vv.height < window.innerHeight * 0.75 (use 0.75 threshold -- mobile keyboards are typically 40%+ of screen)
   - If keyboard just opened (keyboardOpen && lockedHeight === null):
     - Set lockedHeight = window.innerHeight (lock to the FULL height, not the reduced height -- this prevents Phaser ScaleManager from seeing the parent shrink)
     - Set appEl.style.height = `${this.lockedHeight}px`
     - Set appEl.style.overflow = 'hidden'
   - If keyboard just closed (!keyboardOpen && lockedHeight !== null):
     - Set lockedHeight = null
     - Set appEl.style.height = '' (restore to CSS dvh)
     - Set appEl.style.overflow = ''

   private preventPinchZoom(): void:
   - Add 'gesturestart' listener on document: (e) => e.preventDefault(), { passive: false }
   - Add 'gesturechange' listener on document: (e) => e.preventDefault(), { passive: false }
   - Add 'gestureend' listener on document: (e) => e.preventDefault(), { passive: false }
   - Add 'touchmove' listener on document: (e) => { if (e.touches.length > 1) e.preventDefault(); }, { passive: false }
   - Store all listener references for cleanup

   destroy(): void:
   - Remove visualViewport resize listener
   - Remove all gesture/touchmove listeners

   Also export a standalone helper function:
   ```typescript
   export function isMobile(): boolean {
       return /Android|iPhone|iPad|iPod/i.test(navigator.userAgent)
           || window.matchMedia('(pointer: coarse)').matches;
   }
   ```

2. In src/game/main.ts:
   - Import { MobileKeyboardManager } from './systems/MobileKeyboardManager'
   - AFTER the `new Phaser.Game(config)` line, add:
     ```typescript
     const appEl = document.getElementById('app');
     if (appEl) {
         new MobileKeyboardManager(appEl);
     }
     ```
   - The MobileKeyboardManager is a fire-and-forget singleton (no need to store reference since it lives for the entire app lifetime)

Key design decisions:
- Lock to window.innerHeight (not vv.height) when keyboard opens -- this keeps the app at full screen height so the canvas stays the same size, and the keyboard simply covers the bottom portion
- Use 0.75 threshold (not 0.8) to be slightly more aggressive in detecting keyboard open
- The preventPinchZoom() is needed because iOS 10+ ignores user-scalable=no in the viewport meta tag
- The touchmove multi-touch handler prevents two-finger zoom gestures
  </action>

Run `npx vite build` to confirm TypeScript compiles. Grep MobileKeyboardManager.ts for 'visualViewport', 'gesturestart', 'isMobile'. Grep main.ts for 'MobileKeyboardManager'. Confirm the class is exported and instantiated.
  </verify>

MobileKeyboardManager exists and is instantiated in main.ts. It locks #app height when mobile keyboard opens (MOBI-03), prevents pinch-to-zoom via gesture event listeners, and exports isMobile() helper. Phaser ScaleManager will not see parent size changes when keyboard opens.

<task type="auto">
  <name>Task 2: Wire VerbBar into RoomScene lifecycle and add mobile focus guard to TextInputBar</name>
  <files>src/game/ui/TextInputBar.ts, src/game/scenes/RoomScene.ts</files>
  <action>
1. In src/game/ui/TextInputBar.ts:
   - Import { isMobile } from '../systems/MobileKeyboardManager'
   - Modify the focus() method to skip auto-focus on mobile:
     ```typescript
     focus(): void {
         if (!isMobile()) {
             this.inputEl.focus();
         }
         // On mobile, skip auto-focus to prevent keyboard from opening
         // automatically on scene transitions. User taps input to open keyboard.
     }
     ```
   This prevents Pitfall 5 from the research: keyboard opening immediately on every scene transition on mobile.

2. In src/game/scenes/RoomScene.ts:
   - Import { VerbBar } from '../ui/VerbBar'
   - Add a private property: `private verbBar!: VerbBar;`
   - In the create() method, AFTER the TextInputBar is instantiated (look for `this.textInputBar = new TextInputBar(...)`), add:
     ```typescript
     this.verbBar = new VerbBar(gameContainer);
     ```
     where gameContainer is the same container element used for TextInputBar (the #game-container div).
     The VerbBar will be appended after the TextInputBar in the DOM, which is the correct order: canvas -> text parser -> verb bar. This prevents touch event conflicts (Pitfall 7 from research).

   - In the SHUTDOWN cleanup handler (find `this.events.on(Phaser.Scenes.Events.SHUTDOWN, () => {`), add:
     ```typescript
     this.verbBar.destroy();
     ```
     alongside the existing `this.textInputBar.destroy()` call.

   - Find the textInputBar hide() call (if any in other scenes like DeathScene, EndingScene, etc.) -- the VerbBar visibility is handled by CSS media query so no show/hide calls are needed in scene code. The VerbBar is always in the DOM when RoomScene is active; CSS handles showing it only on mobile.

   IMPORTANT: Touch-to-move (MOBI-04) already works. The existing `this.input.on('pointerdown', ...)` handler in RoomScene works identically for mouse clicks and touch taps because Phaser's Input system treats them the same. The `pointer.x`/`pointer.y` are already canvas-local coordinates, and `camera.getWorldPoint()` handles the scale transform. No changes needed for touch-to-move.
