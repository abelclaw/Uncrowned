**RoomScene.ts changes -- add trigger-ending handler following the exact pattern of triggerDeathHandler:**

1. Add a private field declaration near triggerDeathHandler:
   ```typescript
   private triggerEndingHandler!: (endingId: string) => void;
   ```

2. Add the import for EndingSceneData:
   ```typescript
   import type { EndingSceneData } from './EndingScene';
   ```

3. In create(), after the triggerDeathHandler EventBus.on() registration, add:
   ```typescript
   // Ending trigger handler
   this.triggerEndingHandler = (endingId: string) => {
       if (this.isTransitioning) return;
       this.isTransitioning = true;

       // Record ending BEFORE starting EndingScene (same pattern as death recording)
       const meta = MetaGameState.getInstance();
       const isNewEnding = meta.recordEnding(endingId);
       const discoveredCount = meta.getEndingsDiscovered().length;

       // Hide UI elements
       this.textInputBar.hide();
       this.inventoryPanel.hide();

       // Fade out then start EndingScene (full replacement, not overlay)
       this.cameras.main.fadeOut(1500, 0, 0, 0);
       this.cameras.main.once(
           Phaser.Cameras.Scene2D.Events.FADE_OUT_COMPLETE,
           () => {
               this.scene.start('EndingScene', {
                   endingId,
                   isNewEnding,
                   discoveredCount,
                   totalEndings: 4,
               } as EndingSceneData);
           }
       );
   };
   EventBus.on('trigger-ending', this.triggerEndingHandler);
   ```

   Key differences from triggerDeathHandler:
   - Uses scene.start() not scene.launch() (EndingScene replaces RoomScene)
   - Uses 1500ms fadeOut (slightly longer for dramatic effect)
   - Hides textInputBar and inventoryPanel before fade
   - Does NOT use scene.pause() -- RoomScene is fully stopped by scene.start()

4. In shutdown cleanup, add cleanup NEAR the trigger-death cleanup:
   ```typescript
   EventBus.off('trigger-ending', this.triggerEndingHandler);
   ```

**CRITICAL TIMING NOTE (from research Pitfall 1):** The evaluate-ending action MUST be in accept-throne and decline-throne (NOT in perform-rite). If it were in perform-rite, the throne choice flags would not yet be set when the ending is evaluated. By placing it in the throne choice puzzles, both clerk path flags AND the throne flag are set before evaluation.

**PITFALL AWARENESS (research Pitfall 2):** The evaluate-ending action fires synchronously after narrator-say. The narrator-say text starts typewriting in the NarratorDisplay, but PuzzleEngine actions execute synchronously. This means the trigger-ending event fires while the throne narration is still typing. The 1500ms camera fadeOut in the RoomScene handler provides enough delay for the player to see the start of the narration. The full narration text is short (~50-80 words for the throne choices), and the fadeOut is a natural cinematic transition. This is acceptable UX.
