  <action>
    Add a `"puzzleHints"` array to each room JSON file that has hintable puzzles. Place it after `"audio"` (or after the last existing top-level field before `"playerSpawn"`).

    **CRITICAL rules for puzzleHint authoring:**

    1. **Deduplicate alternate syntaxes.** Many puzzles have multiple entries for different phrasings (e.g., `combine-stick-mushroom`, `combine-mushroom-stick`, `use-stick-on-mushroom`). Create ONE puzzleHint entry using the FIRST/primary puzzle ID only. The hint system checks `puzzle-solved:{puzzleId}` flags, and the PuzzleEngine sets flags for ALL variants when any one fires. Pick the most "canonical" ID (usually the shortest/most natural one).

    2. **Skip simple obvious take puzzles.** Items that are plainly described in the room description and have no conditions beyond `item-not-taken` (e.g., `take-stick` in forest_clearing) do NOT need hints. Only add hints for puzzles where the player might genuinely get stuck: multi-step puzzles, conditional takes, use-X-on-Y puzzles, combination puzzles, riddles, and hidden items.

    3. **Voice guide:** The narrator is reluctant, sardonic, condescending, and self-aware. Reference existing death texts and room descriptions for tone.
       - Tier 1 (vague): Coy, withholding. "I suppose I should mention..." style. Hints at the puzzle's existence without naming the solution.
       - Tier 2 (medium): Exasperated. Names the objects or direction involved. "Your ancestors developed opposable thumbs for a reason."
       - Tier 3 (explicit): Bluntly explicit, almost angry at having to spell it out. "Use X on Y. I literally cannot make this simpler."

    4. **Multi-step puzzles must reference prerequisites.** If a puzzle requires items from other rooms, tier 2+ hints should mention where to find them (without naming exact rooms if possible at tier 2, naming rooms at tier 3).

    5. **Rooms with NO hintable puzzles** (e.g., `village_path` which has no puzzles) should NOT get a puzzleHints field at all.

    **Act 1a hintable puzzles (deduplicated):**
    - `forest_clearing`: `take-rusty-key` (hidden in stump crack - hint worthy)
    - `cave_entrance`: `use-key-on-door` (use rusty-key on cave door), `combine-stick-mushroom` (combine stick + mushroom for makeshift torch)
    - `village_square`: `take-toll-coin` (possibly hidden), `read-notice-board` (skip if obvious)
    - `forest_bridge`: `answer-riddle-correct` (troll riddle), `pay-toll` (pay coin to cross)
    - `old_watchtower`: `use-lens-on-telescope` (use lens on telescope to see something)
    - `castle_courtyard`: `show-crystal-to-guards` (show crystal to pass guards), `fill-chalice-at-well` (fill chalice at well), `combine-chalice-sage` (combine chalice + sage for spirit brew)

    **Act 1b hintable puzzles (deduplicated):**
    - `castle_garden`: `take-sage` (may need hints if hidden)
    - `castle_hallway`: `use-skeleton-key` (use skeleton key on door)
    - `royal_kitchen`: `take-chalice` (obvious take - skip unless hidden), `give-flour-to-cook` (give flour to cook)
    - `servants_quarters`: `use-rat-trap` (use rat trap), `take-flour` (may be conditional)
    - `throne_room`: `sit-on-throne` (trigger), `use-spirit-brew` (use spirit brew on throne/ghost), `seal-decree` (combine seal + decree)
    - `treasury`: `take-kingdom-seal` (conditional?), `take-clock-oil` (conditional?)
    - `clock_tower`: `repair-clock` (use gear-spring on clock)

    For each room, read the existing JSON first, then add the puzzleHints array. Check each puzzle's conditions to understand what the player needs -- this informs hint text.

    Keep each hint text to 1-3 sentences. Match the narrator voice from existing descriptions and death texts in the same room.
