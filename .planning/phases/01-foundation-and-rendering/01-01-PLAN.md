<task type="auto">
  <name>Task 1: Scaffold project and configure pixel-perfect game config</name>
  <files>
    package.json
    tsconfig.json
    tsconfig.node.json
    vite.config.ts
    index.html
    public/style.css
    src/main.ts
    src/game/main.ts
    src/game/EventBus.ts
  </files>
  <action>
    Clone the official Phaser Vite TypeScript template into the project root:
    ```bash
    npx degit phaserjs/template-vite-ts . --force
    npm install
    ```

    Then modify the template files to match the KQGame project structure:

    1. **`src/game/main.ts`** -- Create the Phaser.Game config:
       - `type: Phaser.AUTO` (WebGL preferred, Canvas fallback)
       - `width: 960, height: 540` (2x of 480x270 native pixel art resolution, scales cleanly to 1080p and 4K)
       - `parent: 'game-container'`
       - `backgroundColor: '#1a1a2e'` (dark blue-black, fits fantasy adventure)
       - `render: { pixelArt: true }` -- CRITICAL: sets antialias:false, roundPixels:true, nearest-neighbor filtering
       - `scale: { mode: Phaser.Scale.FIT, autoCenter: Phaser.Scale.CENTER_BOTH }`
       - `fps: { target: 60, smoothStep: true }` -- delta smoothing for consistent game speed
       - `scene: [Boot, Preloader, Game]`
       - Export `new Phaser.Game(config)` as default

    2. **`src/main.ts`** -- Entry point that imports `./game/main` to start the game. Keep minimal -- just the import.

    3. **`src/game/EventBus.ts`** -- Create singleton EventEmitter:
       ```typescript
       import Phaser from 'phaser';
       const EventBus = new Phaser.Events.EventEmitter();
       export default EventBus;
       ```
       This is used for cross-scene communication. Do NOT use `this.game.events` (collides with Phaser internal events).

    4. **`public/style.css`** -- Configure pixel-perfect canvas display:
       - `body`: margin 0, padding 0, overflow hidden, background #000
       - `#game-container`: 100vw x 100vh, flex center
       - `#game-container canvas`: `image-rendering: pixelated` AND `image-rendering: crisp-edges` (Firefox fallback)
       This CSS is CRITICAL for pixel art -- without it, the browser applies smoothing.

    5. **`index.html`** -- Ensure the game container div has `id="game-container"` and links to style.css. The template should already have this structure; adjust if needed.

    6. **`vite.config.ts`** -- Keep the template default. Vite handles static assets in `public/` automatically.

    7. **`tsconfig.json`** -- Keep template defaults. Phaser ships full type definitions.

    Delete any template example scenes that were cloned (e.g., template MainMenu, GameOver scenes) -- we replace them with our own Boot/Preloader/Game scenes.
  </action>
  <verify>
    Run `npm run dev` and confirm the dev server starts without TypeScript or build errors. The browser should show the game container with a dark background (no scene content yet since Game scene is a placeholder). Check the browser console for no Phaser errors.
  </verify>
  <done>
    Project scaffolded with Phaser 3.90.0, TypeScript, Vite. Game config has pixelArt:true, Scale FIT, 60fps target. EventBus singleton exists. CSS has image-rendering:pixelated. Dev server runs without errors.
  </done>

  <action>
    Create the three-scene chain (Boot -> Preloader -> Game):

    1. **`src/game/scenes/Boot.ts`** -- Minimal boot scene:
       - Constructor calls `super('Boot')`
       - `preload()`: Empty for now (will load loading-screen assets in future if we use image-based progress bar). For now the progress bar uses Phaser graphics primitives that need no preloaded assets.
       - `create()`: Calls `this.scene.start('Preloader')` to transition immediately.

    2. **`src/game/scenes/Preloader.ts`** -- Loading screen with progress bar:
       - Constructor calls `super('Preloader')`
       - `preload()`:
         - Get camera dimensions: `const { width, height } = this.cameras.main`
         - Create a dark semi-transparent background box (320x50) centered on screen using `this.add.graphics()` with `fillStyle(0x333333, 0.8)`
         - Create a progress bar graphics object (separate from background box)
         - Create "Loading..." text above the bar using `this.add.text()` with monospace font, white color, origin(0.5)
         - Create percentage text centered on the bar, updates to show "0%" through "100%"
         - Listen to `this.load.on('progress', (value: number) => { ... })`: clear and redraw progress bar fill (300 * value width, white fill), update percent text
         - Listen to `this.load.on('complete', () => { ... })`: destroy all loading UI elements
         - Queue placeholder assets for Plan 01-02: create `public/assets/backgrounds/` directory with a `.gitkeep` file. The actual placeholder images will be created in Plan 01-02.
         - For now, to test the loading screen works, load at least one small placeholder: create a simple 1x1 white pixel image programmatically via canvas in the Boot scene and add it as a texture, OR load any small test file. Alternative: the progress bar will still show even with no assets (it just completes instantly) -- this is acceptable for Plan 01-01; Plan 01-02 adds the real assets.
       - `create()`: Calls `this.scene.start('Game')` to transition to the game scene.

    3. **`src/game/scenes/Game.ts`** -- Placeholder game scene:
       - Constructor calls `super('Game')`
       - `create()`:
         - Add centered text "KQGame - Phase 1" using `this.add.text()` to confirm the scene loaded
         - Import and emit on EventBus: `EventBus.emit('scene-ready', { sceneKey: 'Game' })`
         - Log to console: `console.log('Game scene ready')` for debugging
       - `update(time: number, delta: number)`: Empty for now (will be used by parallax in Plan 01-02)
       - Listen for `Phaser.Scenes.Events.SHUTDOWN` to clean up EventBus listeners

    4. **`public/assets/backgrounds/.gitkeep`** -- Create empty file to establish the backgrounds asset directory for Plan 01-02.

    Update `src/game/main.ts` to import all three scenes and list them in the config's `scene` array: `[Boot, Preloader, Game]`.
  </action>

    4. No errors in browser console
