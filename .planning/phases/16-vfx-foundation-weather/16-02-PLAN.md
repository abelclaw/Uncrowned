  <files>src/game/systems/EffectsManager.ts, src/game/scenes/RoomScene.ts</files>
  <action>
1. In `src/game/systems/EffectsManager.ts`, add a public method:
   ```ts
   playInteractionBurst(worldX: number, worldY: number): void
   ```
   - Creates a one-shot particle emitter at the given world position (NOT camera-relative -- these fire at the hotspot location in world space)
   - Uses the 'particle-white' texture
   - Sparkle burst config: quantity: 12, speed: { min: 40, max: 120 }, angle: { min: 0, max: 360 } (radial burst), lifespan: 600, scale: { start: 1.5, end: 0 }, alpha: { start: 1, end: 0 }, tint: 0xffdd44 (golden sparkle)
   - Set depth to 85 (above weather/ambient, visible over room objects)
   - The emitter should fire once (explode: true or emitting: false after initial burst) and auto-destroy
   - Use `emitter.explode(12, worldX, worldY)` for one-shot burst, then `this.scene.time.delayedCall(1000, () => emitter.destroy())` to clean up after particles fade
   - Do NOT setScrollFactor(0) -- this is world-positioned

2. In `src/game/scenes/RoomScene.ts`:
   - Import EffectsManager: `import { EffectsManager } from '../systems/EffectsManager';`
   - Add private field: `private effectsManager!: EffectsManager;`
   - In `create()`, after AudioManager init (line ~456), add EffectsManager lifecycle:
     ```ts
     // Phase 16 effects integration
     this.effectsManager = EffectsManager.getInstance();
     this.effectsManager.init(this);
     this.effectsManager.onRoomEnter(this.roomData);
     ```
   - Wire interactive burst: In the `commandSubmittedHandler`, after `this.commandDispatcher.dispatch()` returns a successful result (non-error), trigger a sparkle burst at the interaction point. Specifically:
     - After `const result = this.commandDispatcher.dispatch(...)`, check if `parseResult.action.subject` resolves to a hotspot with an interactionPoint
     - NOTE: `parseResult.action.subject` is typed `string | null` (the hotspot ID directly, NOT an object). Find the matching hotspot: `const matchedHotspot = this.hotspotZones.find(z => z.hotspot.id === parseResult.action.subject);`
     - If found AND the action verb is NOT 'look' (look is passive observation, no sparkle): `this.effectsManager.playInteractionBurst(matchedHotspot.hotspot.interactionPoint.x, matchedHotspot.hotspot.interactionPoint.y);`
     - Also trigger burst on item pickup: in the `itemPickedUpHandler`, before destroying the sprite, get the sprite position from the `itemSprites` map: `const sprite = this.itemSprites.get(itemId); if (sprite) { this.effectsManager.playInteractionBurst(sprite.x, sprite.y); }` — place this BEFORE the existing `sprite.destroy()` call. The handler only receives `itemId: string`, so use the sprite's world position (not a zone reference)
   - In the shutdown handler, add cleanup BEFORE AudioManager cleanup:
     ```ts
     // Phase 16 effects cleanup
     this.effectsManager.cleanup();
     ```

3. Also trigger burst on click-to-interact: In the pointerdown handler where hotspot click is detected (the `for (const zone of this.hotspotZones)` loop at ~line 372), REPLACE the existing walkTo callback `this.player.walkTo(path, () => this.player.playInteraction())` (line ~378) with an expanded version that adds the sparkle burst:
   ```ts
   this.player.walkTo(path, () => {
     this.player.playInteraction();
     this.effectsManager.playInteractionBurst(
       zone.hotspot.interactionPoint.x,
       zone.hotspot.interactionPoint.y
     );
   });
   ```
   This replaces the existing inline lambda `() => this.player.playInteraction()` — do NOT append a second walkTo call.
