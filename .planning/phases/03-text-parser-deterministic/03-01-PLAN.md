  <name>Deterministic Text Parser with Verb-Synonym Matching and Noun Resolution</name>

  <behavior>
    The TextParser accepts a raw string and room context (hotspots, exits), and returns a ParseResult.

    Parsing pipeline: normalize input -> strip articles/filler -> match verb from synonym table -> extract noun phrases via regex patterns -> resolve nouns against room hotspots/exits/directions -> produce GameAction.

    Test cases (input -> expected output):

    Bare verbs:
    - "look" -> { verb: 'look', subject: null, target: null }
    - "l" -> { verb: 'look', subject: null, target: null }

    Verb + noun:
    - "look at stump" -> { verb: 'look', subject: 'old-stump', target: null } (given room has hotspot id:'old-stump', name:'Old Tree Stump')
    - "examine old tree stump" -> { verb: 'look', subject: 'old-stump', target: null }
    - "take mushroom" -> { verb: 'take', subject: null, target: null } (no mushroom in room -> subject null, success true but subject unresolved)
    - "x stump" -> { verb: 'look', subject: 'old-stump', target: null }

    Two-noun commands:
    - "use key on door" -> { verb: 'use', subject: resolved-or-raw('key'), target: resolved-or-raw('door') }
    - "combine rope and hook" -> { verb: 'use', subject: resolved-or-raw('rope'), target: resolved-or-raw('hook') }

    Go/direction:
    - "go east" -> { verb: 'go', subject: resolved-direction-or-exit, target: null }
    - "go cave" -> { verb: 'go', subject: resolved-exit('to-cave'), target: null } (given exit label matches)
    - "north" -> { verb: 'go', subject: 'north', target: null }
    - "n" -> { verb: 'go', subject: 'north', target: null }
    - "enter cave" -> { verb: 'go', subject: resolved-exit, target: null }

    Article stripping:
    - "look at the old stump" -> same as "look at stump"
    - "take the shiny key" -> strips "the" before noun resolution

    Unrecognized:
    - "xyzzy" -> { success: false, error: "I don't understand 'xyzzy'..." }
    - "" -> { success: false, error: ... }

    Talk commands:
    - "talk to wizard" -> { verb: 'talk', subject: resolved('wizard'), target: null }
    - "ask wizard about sword" -> { verb: 'talk', subject: resolved('wizard'), target: resolved-or-raw('sword') }

    Open/push/pull:
    - "open chest" -> { verb: 'open', subject: resolved('chest'), target: null }
    - "open door with key" -> { verb: 'open', subject: resolved('door'), target: resolved('key') }
    - "push button" -> { verb: 'push', subject: resolved('button'), target: null }
    - "pull lever" -> { verb: 'pull', subject: resolved('lever'), target: null }
  </behavior>
  <implementation>
    Step 1 - Types (pre-TDD setup):
    Create src/game/types/GameAction.ts with:
    - Verb type: 'look' | 'take' | 'use' | 'go' | 'talk' | 'open' | 'push' | 'pull'
    - GameAction interface: { verb: Verb, subject: string | null, target: string | null, rawInput: string }
    - ParseResult interface: { success: boolean, action?: GameAction, error?: string }

    Extend src/game/types/RoomData.ts:
    - Add optional `responses` map to HotspotData: { look?: string, take?: string, use?: string, talk?: string, open?: string, push?: string, pull?: string, default?: string }
    - Add optional `description` to RoomData (room-level look text)
    - Add optional `direction` and `label` to ExitData (for text parser "go east" / "go cave")

    Step 2 - VerbTable:
    Create src/game/parser/VerbTable.ts with VERB_TABLE array of VerbDefinition objects. Each has canonical verb name, synonym list, and regex patterns ordered most-specific-first. Include all 8 verbs plus directional shortcuts.

    Step 3 - NounResolver:
    Create src/game/parser/NounResolver.ts. Resolves a raw noun string against room hotspots and exits.
    Resolution order: exact ID match -> exact name match -> partial word match -> direction mapping -> exit label/targetRoom match.
    Returns ResolvedNoun { type: 'hotspot'|'exit'|'direction'|'unknown', id: string, confidence: 'exact'|'partial'|'none' }.
    For unresolved nouns, return type:'unknown' with the raw string as id (Phase 4 inventory will extend this).

    Step 4 - TextParser:
    Create src/game/parser/TextParser.ts. The parse(input, hotspots, exits) method:
    1. Normalize: trim, lowercase
    2. Match verb: iterate VERB_TABLE, test each pattern against input
    3. Extract noun phrases from regex capture groups
    4. Strip stop words from noun phrases
    5. Resolve nouns via NounResolver
    6. Return ParseResult with GameAction

    Step 5 - Tests (RED -> GREEN):
    Install vitest (check if already available). Create test file covering all test cases above.
    Run tests (expect failures), implement/fix code, run tests (expect passes).

    IMPORTANT: The NounResolver should return the raw noun string (type:'unknown') for nouns that don't match anything in the current room -- the CommandDispatcher (Plan 02) handles the "you don't see that here" response. This allows "use key on door" to parse successfully even if "key" is an inventory item not in the room context. Phase 4 inventory will extend NounResolver to check inventory.
  </implementation>
