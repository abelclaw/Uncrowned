  <action>
Create `src/game/systems/QualitySettings.ts`:
- Export `type QualityLevel = 'high' | 'low' | 'off'`
- Singleton class `QualitySettings` with `getInstance()`, `resetInstance()` for testing
- Private `level: QualityLevel` defaulting to 'high'
- `getLevel(): QualityLevel` getter
- `setLevel(level: QualityLevel): void` setter that saves to localStorage key `kqgame-quality`
- Constructor reads from `localStorage.getItem('kqgame-quality')` with try/catch, falls back to 'high'
- `isCanvasRenderer(game: Phaser.Game): boolean` static method checking `game.renderer.type === Phaser.CANVAS` (import Phaser)
- Mobile detection: `isMobileDevice(): boolean` static method using existing `isMobile()` from `../ui/VerbBar` or re-implement: `'ontouchstart' in window && window.matchMedia('(pointer: coarse)').matches`

Modify `src/game/systems/EffectsManager.ts`:
- Import QualitySettings at top
- Add private method `getQualityMultiplier(): number` that returns:
  - 0 if quality is 'off'
  - 0 if renderer is Canvas (check via `this.scene.game.renderer.type === Phaser.CANVAS`)
  - 0.5 if quality is 'low' OR if `QualitySettings.isMobileDevice()` AND quality is 'high' (mobile auto-downgrades to low particle counts)
  - 1.0 if quality is 'high' and not mobile and not Canvas
- In `onRoomEnter()`: check `getQualityMultiplier() === 0` early and skip all effect creation (weather, ambient, lighting). Still call clearAll() so previous effects are cleaned up.
- In `createWeatherEmitter()`: multiply quantity by `getQualityMultiplier()`. If result rounds to 0, skip emitter creation.
- In `createAmbientEmitter()`: same quantity scaling.
- In `applyLighting()`: skip PostFX (bloom, glow, desaturate, vignette) if quality is 'off' or renderer is Canvas (PostFX requires WebGL). Still apply the lighting overlay Rectangle (it works on Canvas). Skip torchFlicker timer if quality is 'off'.
- In `playInteractionBurst()`: scale burst quantity (12) by multiplier. If 0, skip burst entirely.

Modify `src/game/scenes/RoomScene.ts`:
- No changes needed for basic integration since EffectsManager reads QualitySettings internally.
- But add a one-time Canvas detection log in create(): `if (this.game.renderer.type === Phaser.CANVAS) console.log('[RoomScene] Canvas renderer detected, effects reduced');`

Important: Do NOT import isMobile from VerbBar (circular dependency risk). Re-implement the mobile check in QualitySettings or extract to a shared utility if one exists. Check if `src/game/utils/` has a mobile detection utility first.

  - After cycling, call `QualitySettings.getInstance().setLevel(newLevel)`

  - Store reference to the text object so it can update in place

  ```
  const qualityLevels: QualityLevel[] = ['high', 'low', 'off'];
  let currentIndex = qualityLevels.indexOf(QualitySettings.getInstance().getLevel());
  const qualityText = this.createMenuItem(`Quality: ${capitalize(qualityLevels[currentIndex])}`, 480, y, () => {
      currentIndex = (currentIndex + 1) % 3;
      const newLevel = qualityLevels[currentIndex];
      QualitySettings.getInstance().setLevel(newLevel);
      qualityText.setText(`Quality: ${capitalize(newLevel)}`);
  });
  ```
